Creating an Account

Creating a HARP account is required to log in and access iQIES. Your HARP account can be used to log into one or many CMS applications.

Creating a HARP Account

  1. Register for a HARP account at https://harp.qualitynet.org/register
    • HARP is a secure identity management portal that verifies your identity by taking personal information, such as Date of Birth and SSN, so that Experian can generate a list of personal questions for you to answer to verify your identity.
  2. Enter your Profile Information and click "Next".
    • If you do not wish to enter your SSN, you may "Initiate Manual Proofing". It is not recommended to initiate manual proofing if you can complete remote proofing, as it typically takes longer to be proofed and verified.
  3. Enter your Account Information and click "Next".
  4. If remote proofing questions were successfully generated, answer the questions and click "Next".
    • If you were unable to generate remote proofing questions OR you answered your remote proofing questions incorrectly, you have two options:
      • If you think you know what you entered incorrectly, you can "Retry Remote Proofing" back on the Profile Information page.
      • Call Experian to verify your identity over the phone and then "Enter Reference Number" back on the Profile Information page.
  5. Upon account creation, you are automatically set up with email two-factor authentication. If you would like to add additional two-factor authentication devices, click "Log into HARP" on the Confirmation page. Otherwise, go directly to iQIES to request a role.

Requesting Roles

You will not have access to any functionality in iQIES until you have at least one approved user role. iQIES user roles are requested in iQIES and must be approved by the organization's Security Official.

Requesting Roles in iQIES

  1. To request a role in iQIES, log into iQIES at https://iqies.cms.gov with your HARP user ID, password, and two-factor authentication.
  2. Click on the "Request a Role" button to begin the role request process.
  3. Select your user category and click "Next". To learn more about the differences between user categories, view the User Roles Matrix.
  4. Select your user role and click "Next". To learn more about the differences between user roles, view the User Roles Matrix.
  5. Select the organizations that you need to access and click "Submit".
  6. You should see the requested roles on your Pending Requests list. You will receive an email notification to let you know once your request has been approved or rejected.
    • The time it takes for a request to be approved or rejected varies depending on the Security Official and when they get around to handling the request.
  7. Upon role approval, you may log into iQIES to access the associated functionality and permissions.

Managing ID and Password

Log into HARP to update your password, profile information, challenge question, and manage two-factor authentication devices. It is not possible to change your user ID.

Forgot Your User ID

If you have forgotten your user ID, do the following:

  1. Go to HARP at https://harp.qualitynet.org and click "Having trouble logging in?"
  2. Select "Forgot User ID or Password".
  3. Enter your email address and click "Send Email".
  4. You should receive an email that contains your user ID - you may ignore the "Reset Password" button if you do not need to reset your password.

Forgot Your Password

If you have forgotten your password, do the following:

  1. Go to HARP at https://harp.qualitynet.org and click "Having trouble logging in?"
  2. Select "Forgot User ID or Password".
  3. Enter your email address and click "Send Email".
  4. You should receive an email that contains your user ID and a button to "Reset Password".
  5. Click on "Reset Password" - you should be prompted to answer your challenge question.
  6. Answer your challenge question and click "Next".
    • If you forgot your challenge question answer, follow the instructions below for 'Forgot your password and challenge question'.
  7. Enter your new password and confirm your new password, then click "Reset Password".
  8. You should now be able to log in with your user ID and new password.

Forgot Your Password and Challenge Question

If you have forgotten your challenge question, you will be unable to reset your password. To initiate a full account recovery, do the following:

  1. Go to HARP at https://harp.qualitynet.org and click "Having trouble logging in?"
  2. Select "Full Account Recovery".
  3. Enter your email address, first name, last name, and date of birth and click "Send Email".
  4. You should receive an email that contains your user ID and a button to "Recover Account".
  5. Click on "Recover Account" - you should be prompted to select a challenge question and answer.
  6. Select a challenge question and answer, then click "Reset Challenge Question".
  7. Enter your new password and confirm your new password, then click "Reset Password".
  8. You should now be able to log in with your user ID and new password.

Managing My Profile

​The My Profile page contains your User Information, Pending Requests, and User Roles.

User Information - Pulled from HARP and includes your user ID, email address, and phone number (if available).
Pending Requests - Lists your pending role requests. To request additional roles, click on the "Request User Role" button.
User Roles - Lists your approved iQIES user roles which determine your access and permissions within the application. Some user roles will have an "Edit" button that allows the user to modify role-related information, such as Federal Surveyor ID, Management Unit, and Work Unit.

Edit your User Information

If you would like to update your User Information, you will need to edit it in HARP. It is not possible to change your user ID.

  1. To edit your User Information, log into HARP at https://harp.qualitynet.org
  2. You should land on your User Profile. Click the "Edit" button to update your email address, home address, or phone number.
    • You will need to contact the Help Desk to update your First Name or Last Name.
  3. Click "Save".
    • If you are updating your email address, you will be prompted to enter a code sent to your email to verify your address before your changes will be saved.
  4. Your information should be successfully updated and reflected in iQIES shortly.

Role Approvals

If you are an iQIES Security Official that needs to manage roles and permissions, log into HARP to approve, reject, and remove user roles.

Approve/Reject Role Requests

  1. If you are a Security Official, log into HARP at https://harp.qualitynet.org
  2. Click on the "Security Official" nav item in the top right.
  3. You should land on the Role Requests screen and see a list of pending role requests across Organizations that you are a Security Official for.
  4. To approve one or many requests, select the checkbox per role request and click "Approve".
  5. To reject one or many requests, select the checkbox per role request and click "Reject".
    • You will be prompted to enter a Rejection Reason - if you have selected more than one role request to reject, the same Rejection Reason will be used for all of them.
  6. The user should receive an email notification upon role approval or rejection.

Remove User Roles

  1. If you are a Security Official, log into HARP at https://harp.qualitynet.org
  2. Click on the "Security Official" nav item in the top right.
  3. Select the "User Lookup" nav item on the left.
  4. Select your Program and Organization - you should see a list of all users in the selected Organization.
  5. Use the search bar to search for users by first name, last name, email, or user ID.
  6. Click "View" per user to see their Profile Information and user roles.
  7. Click on the "Remove" link for the user role you would like to remove.
  8. Confirm the removal.
  9. The user's role should be successfully removed.

Managing Patient Profiles

How to Add a New Patient

  1. From the Assessments landing page, select Add Patient.
  2. Enter the patient's information.
  3. Select Add Patient to finish.

How to Modify or Delete Existing Patient Information

  • Modify an existing patient record by selecting Edit. When you are finished editing, scroll to the bottom of the page and select Update Patient Profile.
  • Delete a Patient's profile by scrolling to the bottom of the page and selecting Delete Patient Profile .

Note: Updates to a patient’s profile will be reflected only in their iQIES profile, not in any submitted or in-progress assessments. To make updates to a patient’s information within an assessment, it will need to be inactivated, updated, and resubmitted.

Uploading Assessments

How to Upload an Assessment

  1. Navigate to the Assessments landing page.
  2. Select Upload Assessments and choose the .zip files you want to upload.
  3. Then select Open, Enter or Save (depending on your browser ).
  4. You will receive a Success notification when the files have been successfully uploaded. To view the report, select View Report. If View Report isn’t activated, the report has not been processed into iQIES. Wait a few moments and refresh the page.

How to Correct Rejected Assessments

If an uploaded assessment is rejected with errors, edit the errors by doing the following:

  1. From the Assessments Upload page, select View Report from the submission history. Submissions with a status of "Rejected" means that there are one or more assessments with errors.
  2. Document the errors.
  3. Return to the vendor tool or the file and fix the errors, then export and upload the rejected assessments again.

Creating Assessments

How to Create a New Assessment

  1. Navigate to the desired patient’s information through the Assessments landing page.
  2. Select Create New Assessment.
  3. Fill out the assessment and select Save and Continue to save each section of the assessment. You can return to the assessment at any time to complete or submit.
  4. When all the required information has been entered you will be able to submit the assessment. If there is an error, the Submit Assessment button will be grayed out until all the information has been correctly entered.
  5. To verify if an assessment has been accepted or rejected, navigate to the patient’s profile page and check the status column in the Assessments table.

Modifying / Inactivating Assessments

How to Modify Assessments

If an assessment contains inaccurate information, it is possible to modify some information.

  1. Navigate to the patient’s profile page and select Actions , then View Assessment for the assessment that needs to be modified or open the web view of the assessment.
  2. Select Modify.
  3. Update the desired information.

Note: Some information within the assessment cannot be modified and can only be changed by inactivating and resubmitting the assessment.

How to Inactivate an Assessment

Assessments with errors that cannot be corrected with a modification need to be inactivated. To inactivate an assessment:

  1. Navigate to the patient’s profile page and select Actions, then View Assessment for the assessment that needs to be modified or open the web view of the assessment.
  2. Select Inactivate.

Running Reports

Reports are NOT automatically saved to My Reports. To save a report, you must select Save to My Reports on the web view of the report.

How to Find and Run a Report Type

  1. Navigate to the Reports landing page.
  2. Select Find Report Type.
  3. Enter a keyword or select the report category or report type from the drop-down menu.
  4. Select Run Report.

How to Run a Frequently Run Report

  1. From the Reports landing page, select Run Report from your Frequently Run Report Types to navigate to the Run Report page.
  2. Enter the required report criteria, such as Begin Date, End Date, or Provider Name.
  3. Select Run Report.
  4. Upon running a report, you will automatically be redirected to the web view of the report.

How to Re-Run a Report

From the My Reports page, select Run Again from the drop-down. It will navigate to the Run Report page for the selected report type. The report filters will be pre-populated with the same criteria used to run the original report

How to Search for Providers

  1. In the provider search field, enter the name, address, or CCN of a provider.
  2. Select the provider from the available drop-down options. One or multiple providers may be selected.

How to Save Report Criteria

  1. Enter the required report criteria.
  2. Select Save Criteria

How to Schedule a Report Run

  1. Navigate to the Run Report page and enter the required report filters.
  2. Select the Schedule Report Run checkbox.
  3. Enter the date and time for your scheduled report to be run.
  4. Select Run Report. A success notification will confirm the report was successfully scheduled.

Viewing, Downloading, and Saving Reports

How to View a Report

  1. Select View My Reports from Reports landing page.
  2. Choose the name of the report or select the More drop-down.
  3. Select View.

How to Download a Report

The two possible downloadable file formats are CSV and PDF. All reports have a CSV option for download. Some reports will also include a PDF option. You can download reports from the web view or from My Reports .

  1. In the web view of the report, select the Download drop-down in the top-right and choose the file format you would like to download. The file will start downloading to your computer.
  2. In My Reports, select the More drop-down for the report. Then select Download CSV or Download PDF.

How to Rename a Report

  1. From the My Reports page, select the More drop-down next to the desired report.
  2. Select Rename. The name of the report will be replaced with an input field for you to modify the name.
  3. Select Save.

How to Save a Report

After running a report, you will be redirected to the web view.

  1. Select Save to My Reports in the top-right.
  2. You will be prompted to name your report.
  3. After naming your report, select Save.

How to Navigate the Web View of a Report

The web view (HTML) provides an in-browser dynamic view of the report. The functionality available on the web view of the report includes:

  • Report Actions
    • Save to My Reports - Prompts you to name and save the report.
    • Download - Allows you to download the report in available file formats, such as CSV and PDF.
  • Expand/Collapse Containers
    • Report data is displayed within expand/collapse containers so that unwanted information can be hidden. Information can be expanded/collapsed at the provider level as well as at the table level if the report contains multiple tables per provider.
  • Table Filtering
    • Table column filters allow the data to be filtered directly within the report. You can use one or many column filters to filter the table.
  • Table Sorting
    • Table columns can be sorted ascending/descending by selecting the table header.

How to Add and Manage Provider Information

To add a new provider, select Add a Provider from the Providers section of the menu or the Providers landing page. Begin by filling out the basic information then select Add Provider.

Additional provider information can be added or edited by selecting View Provider Details on the Provider Summary section of the Provider’s Profile page. Provider details are organized into the following sections:

  • Basic Information
    • The overview of the provider including name, provider type, and primary location address.
    • Once the Basic Information is entered, the remaining sections will be enabled.
  • Mailing Address
    • The mailing address, if it is different from their primary practice location.
  • Additional Branch Addresses
    • Separate branch offices, affiliate facilities, sites, and other types of locations.
  • Operation Details
    • Provider operating status, important dates, and special operating hours.
  • Additional Contacts
    • Emergency contacts and any other contact information.
  • Certification and Licensure
    • Federal certification status, licensure information, CCN and other identification numbers.
  • Deeming Information
    • Deeming status and accrediting organization details.
  • Administrators
    • All administrators and their contact information.
  • Letters
    • Uploaded letters related to the provider.
  • Attachments
    • All supplemental documentation for the provider, including attachments for surveys, complaint and incident intakes, and enforcements.

Find a Provider

Providers can be found by entering their name, CCN, state facility ID, or iQIES ID into the search on the Providers landing page. An advanced search is available for more detailed criteria.

How to Add and Manage Surveys

To create a new survey, navigate to the Provider’s Profile page and select Add Survey.

Existing surveys can be viewed and edited by locating the desired survey in the Recent Surveys table or the View All Surveys history table. Survey details are organized into the following sections:

  • Basic Information
    • The overview of a survey, including the type, survey category, intakes to include, regulation sets, start and exit dates, and survey status.
    • Once the Basic Information is entered, the remaining sections will be enabled.
  • Responsible Staff
    • Where the staff members who work on the survey are added and managed.
    • When a user performs certain functions within the survey their name may be added as a responsible staff member.
  • Survey Team
    • Survey team members can be added from the list of available staff members or can be found through a search.
    • The survey team coordinator will default to the first staff member added to the team and can be changed, if necessary.
  • Citations and Plan of Corrections (POC)
    • Citation tags are added by using the search function and a full list is prepopulated below the search bar.
    • All survey team members can add citations, the Deficient Practice Statement, and citation text.
    • After a citation is added, surveyors can add the Deficient Practice Statement and citation text. Additionally, federal citations can be connected to state citations.
    • Completed citation text can be indicated by marking the Writing Complete checkbox.
    • If multiple surveyors add findings on a citation, and once all citations are marked Writing Complete, the citation text can be merged by the team coordinator.
    • Once citations are merged by the team coordinator, they can be locked. If additional edits are needed, citations can be unlocked and revised.
    • After the Statement of Deficiencies has been sent and the Plan of Correction has been received, the dates and status of the Plan of Corrections can be entered and updated.
    • Plan of Correction dates can be bulk applied to all citation tags or they can be entered individually.
  • Statement of Deficiencies
    • The Statement of Deficiencies (CMS-2567) can be generated and downloaded to be viewed as a draft or as the final version to be sent to the provider.
    • The date the Plan of Correction is due back from the provider is calculated by the Date Due to the Provider and the Date Sent.
  • CMS-1572
    • For HHA providers, branch, staff, and additional services information can be updated.
  • CMS-670
    • Both team and individual survey hours can be recorded within the CMS-670. Individual hours are recorded into the MyTime section and team hours are recorded within Team Time.
  • Letters
    • Uploaded letters related to the survey.
  • Notes
    • Notes can be created and are available for other users to view.
  • Attachments
    • All supplemental documentation for the survey.

State API Overview

This State API documentation currently applies to the training environment, but will apply to the production environment in the future.

An Application Programming Interface (API) is a communication method between a client and a server. It allows systems to share data with each other through specific protocols. APIs work by sending requests for information from a web application or web server and receiving a response. If the client makes a request in a specific format, it will always get a response in a specific format or initiate a defined action.

The iQIES State API service bridges the gap between a state's data needs and the iQIES system. By supporting standards-based data formats and protocols, the iQIES State API connects state systems (clients) to iQIES (server) to support state-specific applications and reporting demands.

View examples in the Github repository.

Available Data

Upon each release of S&C functionality and the migration of a specific Provider Type (e.g. Home Health Agency) into iQIES, the survey and certification data for that Provider Type will be migrated permanently to the central iQIES database and thus will no longer be maintained/updated in state-level databases.

The API will provide states access to the following data:

  • Provider demographics and certification data
  • Survey and citations
  • Complaints and incidents
  • Enforcement cases
  • Personnel/User information
  • State work and management units (if exists)

Getting Started

API Basics

Base Request URL: https://iqies.cms.gov/api/stateapi/

The iQIES State API service builds upon RESTful API standards thus you can choose to consume the iQIES API with any client once you've generated an API key. You can choose to use a RESTful client such as Postman, Insomnia, Swagger or modify your ODBC driver for State API Service using an SDK such as OpenAccess SDK.

API Key

The iQIES API uses API keys to authenticate and authorize API requests. States are authorized only to pull(read) your state data, and State Agency users with role "State Agency API Key Manager" have the necessary permissions to generate the API Key for your state in iQIES.

For the purposes of the training environment you will be assigned an API Key.

In a future production environment you will generate your own API key if you have the right role and an email notification is sent to you when you generate your iQIES API Key. A system-generated email will notify you about pending and current expiration of your API key. There will be three email notifications to alert you about the pending expiration of your API key (15 days, 5 days, and 1 day) and one sent upon expiration.

State API Syntax

The core use case of the iQIES API is to support applications created and run by a state's Departments of Health that currently depends on data retrieved from a direct ODBC/Oracle TNS client connections to state-level ASPEN Oracle databases.

SQL Query Syntax

The queries you execute are ANSI Standard SQL Query Syntax and not a specific flavor of SQL. The SQL itself is sent to the API wrapped in a JSON formatted body.

The new database technology that iQIES uses and which will be hosting the State API endpoint is Postgres, and Postgres’ flavor of SQL is called PL/pgSQL. It is similar to Oracle, and in fact was developed/adapted directly from Oracle, but it is not 100% compatible.

Oracle’s engine is much older, and Oracle’s PL/SQL language includes a very rich set of aggregate functions and Oracle-only extensions that do not exist in any other database engine. These extensions are not necessarily supported in Postgres. The best bet is to stay with ANSI-compliant SQL as a starting point; both Oracle and Postgres support ANSI standard SQL for certain. Postgres does include support for commonly used aggregate functions that may be in frequent use (Denserank, partition over, etc) but will not support Oracle-specific things like rownum, use of the dual table, NVL(), embedded regular expressions, listagg functions, etc.

There are some Oracle to Postgres conversion tools available and the internet is rich with examples of organizations who have migrated from Oracle to Postgres and have posted about your experiences converting code.

Unsupported SQL Queries

The following types of queries are not supported:

  • Any query with RIGHT JOIN.
    • Note: LEFT JOIN and INNER JOIN are supported.
  • If a query has transaction tables and uses GROUP BY and HAVING, where the HAVING clause doesn't have a primary key from the left-most transaction table.
    • Note: Queries without transaction tables using GROUP BY and HAVING will work. Queries with transaction tables using GROUP BY only will also work.
  • Any query which has used "vw" as table alias. "vw" is reserved word for us and any query should not have use this word as table alias.
  • Any query without schema Qualified Table Name.
  • Tables with reserved keywords in our Postgresql DB will need to have a table alias name in order for the query to be executed successful. Reserved keywords can be found here: https://www.postgresql.org/docs/9.4/sql-keywords-appendix.html

Endpoints Overview

Below are the list of endpoints and consist of example queries when applicable. The State API service will construct the SQL query to fetch the data and will always include this state value as one of the filters in the where clause.

Method Endpoint Usage
GET /v1/tables Gets a list of tables in the schema.
GET /v1/tables/{tableName}/cols Gets the columns and constraints information for a pre-existing table.
POST /v1/rows This endpoint will accept a SQL request in JSON format and return results of the SQL query. The SQL request is a JSON with the keys being the keywords in the SQL statement and the values being the column names, table name, etc as the case may be. Response will be limited to 500 rows and requests can be made to fetch additional rows using the offset key in the JSON request. The order of the keys in the JSON request must be in the same order as expected when writing a SQL statement. In addition to the data results, the response will also include an error object as well the actual SQL statement that was executed based on the key-values in the JSON request.

Status Codes & Errors

HTTP Status Codes

Code Description
200 - OK Everything worked as expected.
400 - Bad Request The request was unacceptable, often due to missing a required parameter.
401 - Unauthorized No valid API key provided or if an API key is not supplied.
403 - Forbidden The API key is expired, or does not have permissions to perform the request.
500 - Server Errors Something went wrong on iQIES end.

Error Types

Error Type Description
missing JOIN key Missing JOIN key
invalid JOIN Invalid JOIN. Only left and inner joins are allowed.
authentication_error Failure to properly authenticate yourself in the request.
missing schema Please provide the schema name to qualify the table name. Valid syntax: schemaName.tableName or schemaName.tableName as aliasName or schemaName.tableName aliasName.
multiple table Specify only the schema qualified table name in the FROM key. Use joins to specify any joins.
rate_limit_error Too many requests hit the API too quickly or the amount of rows you are trying to pull exceed the limit (500) rows.
missing primary key If sql has GROUP BY, HAVING clause and any transaction table then HAVING clause must include primary key from the left-most transaction table.
invalid usage of table alias "VW" Any query which has used vw as able alias. vw is reserved word for us and any query should not have use this word as table alias.
invalid usage of table alias "USER" Any query which has used user as table without alias. user is reserved word for us and any query should not have use this table without table alias.
database exception Syntax error. Please review and correct the syntax error in the query above.
